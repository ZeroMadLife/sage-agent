# Phase 6.1：框架抽象层 — AgentRuntime + Skill

> 日期：2026-07-05
> 状态：实现完成，已通过全量验收

## Goal

把 TourSwarm 从“旅游助手实现”向“个人 Agent 框架”推进一步：保留旅游侧行为，把主 ReAct 循环抽象成通用 `AgentRuntime`，并把旅游 prompt、工具集、多 Agent 图、验证器、记忆策略封装为 `travel-planning` Skill。

## 范围边界

- 做：`core/skill.py`、`AgentRuntime` 注入 Skill、`Verifier` 接口、长期记忆 scope、启动入口注册 `travel-planning`。
- 不做：coding Skill、RAG、Dify/Coze、前端 Skill 切换、旅游 MCP 或多 Agent 图内部重写。
- 回归红线：旅游侧 ReAct 行为、工具调用、流式事件、API 启动路径保持兼容。

## 实际改动

### 1. Skill 抽象

新增 [core/skill.py](/Users/zeromadlife/Desktop/tour-agent/core/skill.py)：

- `ToolSpec`：工具名、描述、handler、参数元数据。
- `MemoryConfig`：Skill 记忆策略配置。
- `KnowledgeSource`：后续 RAG 知识源占位。
- `Skill`：完整助手人格，包含 prompt、tools、sub_agent_graph、verifier、memory_config、knowledge_sources。
- `SkillRegistry`：按 name 注册和查找 Skill。
- `build_travel_planning_skill(...)`：把现有旅游 prompt、旅游工具、LangGraph sub-agent graph、`ItineraryVerifier` 打包成 `travel-planning`。

### 2. AgentRuntime

修改 [agents/react_agent.py](/Users/zeromadlife/Desktop/tour-agent/agents/react_agent.py)：

- `TourAgent` 抽象为 `AgentRuntime`。
- 构造函数支持 `skill=Skill`，从 Skill 读取 `system_prompt` 和 `tools`。
- 保留 `TourAgent = AgentRuntime` alias，旧测试和旧调用 `TourAgent(llm, tools=...)` 仍然可用。
- ReAct 循环、工具降级、天气强制工具调用、历史天气复用、行程提取逻辑不改。

### 3. Verifier 接口

修改 [core/verifier.py](/Users/zeromadlife/Desktop/tour-agent/core/verifier.py)：

- 新增 `Verifier` Protocol。
- 新增 `ItineraryVerifier`，内部包装原 `verify_itinerary(...)`。
- `VerificationResult` / `VerificationIssue` 保持原结构，避免 API 和 eval 连锁变更。

### 4. 长期记忆 scope

修改 [core/memory/long_term.py](/Users/zeromadlife/Desktop/tour-agent/core/memory/long_term.py) 和 [core/memory/extractor.py](/Users/zeromadlife/Desktop/tour-agent/core/memory/extractor.py)：

- `LongTermMemory.search(..., scope=None)` 默认 `user:{user_id}`。
- `LongTermMemory.extract_and_store(..., scope=None)` 默认 `user:{user_id}`。
- 写入 Mem0 时带 `metadata={"scope": scope}`。
- 检索 Mem0 时带 `filters={"scope": scope}`。
- `MemoryManager` 默认 scope 为 `skill:travel-planning`，旅游偏好默认不污染后续 Skill。

### 5. API 启动接入

修改 [api/main.py](/Users/zeromadlife/Desktop/tour-agent/api/main.py)：

- 启动时构建旅游工具 dict。
- 用 `SkillRegistry` 注册 `travel-planning`。
- 从 registry 取 Skill 后传入 `AgentRuntime`。

修改 [api/routes.py](/Users/zeromadlife/Desktop/tour-agent/api/routes.py)：

- `SessionState` 增加 `skill_name="travel-planning"`，为后续 Skill 切换留出会话绑定点。
