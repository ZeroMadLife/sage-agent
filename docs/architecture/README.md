# TourSwarm 架构图

> 原图来自 KimiSwarm 研究报告，存放在本目录。

## 多Agent协作架构（Supervisor模式）

![多Agent协作架构](agent_collaboration_flow.png)

4个专业Agent通过Supervisor协调：规划Agent生成行程框架 → 推荐Agent填充景点 → 预算Agent校验约束 → 信息Agent获取实时数据。Supervisor负责意图识别、任务分解、结果汇总与冲突消解。

## 用户交互流程

![用户交互流程](user_interaction_flow.png)

五步模型：对话式输入 → 意图识别 → Agent协作 → 结构化输出 → 用户确认。支持一键调整（换景点/压缩预算/加减天数），每次调整触发反馈循环。

## 10周开发时间线

![开发时间线](development_timeline.png)

5个Phase（每2周一个），5个里程碑（M1-M5）。遵循"前置风险"原则：核心架构在早期完成，为后期预留缓冲。
