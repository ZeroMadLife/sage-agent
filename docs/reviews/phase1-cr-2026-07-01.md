## 代码质量评估

### ✅ 做得好的地方

**1. 类型安全 — 比计划要求做得更好**

Codex 不只是用了 type hints，还把所有返回类型从 `dict` / `list[dict]` 升级为了 **TypedDict**：

```python
# 计划里写的是 list[dict]
# Codex 实际实现：
class AmapPoi(TypedDict):
    id: str
    name: str
    location: str
    rating: str
    ...

async def search_attractions(...) -> list[AmapPoi]:  # 精确类型
```

`RouteMode = Literal["walking", "driving", "transit"]` 也是加分——用 Literal 限定取值范围，比 `str` 更安全。这正是我们选 Python + mypy 想要的效果。

**2. 防御性编程 — 对外部 API 响应做了 isinstance 校验**

```python
pois = data.get("pois", [])
if not isinstance(pois, list):  # 防止高德返回非预期结构
    return []
return [self._normalize_poi(poi) for poi in pois if isinstance(poi, dict)]
```

每个从外部 API 拿到的字段都做了类型校验，这在生产环境中非常重要——高德/和风的 API 偶尔会返回异常结构。

**3. 测试质量 — 覆盖了 happy path + 错误路径 + 委托验证**

```
test_search_attractions_returns_pois          ← happy path
test_search_attractions_handles_empty_response ← 空结果
test_search_attractions_raises_on_api_error    ← API 错误
test_search_attractions_tool_delegates_to_client ← 委托验证（mock client）
```

Server 测试不仅验证工具存在，还用 AsyncMock 验证了工具确实委托给了 client，且参数传递正确。这是优秀的测试设计。

**4. tenacity 重试配置加了 `reraise=True`**

```python
@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, max=10), reraise=True)
```

计划里没要求这个，但 Codex 主动加了——确保重试耗尽后抛出原始异常而非 RetryError，方便上层捕获和处理。

**5. Mock 数据真实度高**

`amap_poi_search.json` 的结构完全模拟了高德真实响应（status/count/pois/biz_ext），不是随便编的假数据。这样后续对接真实 API 时不会有结构偏差。

### ⚠️ 建议优化项（非阻塞，可在 Phase 2 前或期间修复）

**1. `geocode` 方法缺少 `@retry` 装饰器**

```python
# amap/client.py — search_attractions 和 get_route 都有 @retry
@retry(stop=stop_after_attempt(3), ...)
async def search_attractions(...): ...

@retry(stop=stop_after_attempt(3), ...)
async def get_route(...): ...

# 但 geocode 没有
async def geocode(self, address: str, city: str = "") -> GeocodeResult:  # ← 缺 @retry
```

**影响：** geocode 调用失败时不会自动重试，与其它两个方法行为不一致。
**建议：** 加上 `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, max=10), reraise=True)`。

**2. transit 路线规划的 city 参数硬编码为"杭州"**

```python
# amap/client.py:104
if mode == "transit":
    params["city"] = "杭州"  # ← 硬编码，非杭州城市会出错
```

**影响：** 公交路线规划只对杭州有效，用户查北京的公交路线会失败。
**建议：** `get_route` 方法增加可选参数 `city: str = ""`，transit 模式时由调用方传入。Phase 2 的规划 Agent 调用时已有目的地信息，可以传入。

```python
async def get_route(
    self, origin: str, destination: str, mode: RouteMode = "walking", city: str = ""
) -> RouteResult:
    ...
    if mode == "transit":
        params["city"] = city or "杭州"  # fallback
```

**3. `core/mcp_client.py` 覆盖率 0%**

```python
# core/mcp_client.py — build_config_from_settings 未被测试
def build_config_from_settings(
    settings: Settings, scenic_data_path: str = "data/mock/scenic_spots.json"
) -> McpConfig:
    return build_mcp_config(
        amap_api_key=settings.amap_api_key,
        ...
    )
```

**影响：** 这个函数是 Settings → MCP 配置的桥梁，Phase 2 会用到。当前没有测试。
**建议：** 加一个简单测试验证它能从 Settings 构建出正确的配置。非阻塞，Phase 2 开发时补即可。
