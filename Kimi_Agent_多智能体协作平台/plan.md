# 多Agent协作项目研究 & 产品方案设计 - 执行计划

## 目标
输出一份完整的研究报告：涵盖技术栈调研、竞品分析、产品方案设计，帮助用户完成一个"有区分度 + 可落地 + 适合秋招"的多Agent协作项目。

## Stage 1 — 深度研究（Deep Research）
加载技能：`deep-research-swarm`

### 研究方向（并行）
1. **技术栈调研** — MCP协议、LangChain/LangGraph、多Agent协作框架、上下文管理、记忆管理方案
2. **参考项目分析** — 分析 awesome-llm-apps 和 hello-agents 中的优秀案例
3. **竞品/对标产品** — 调研市面上可上线的Agent工具（个人提效、旅游助手等方向）
4. **秋招项目区分度分析** — 什么样的Agent项目在秋招中能让面试官眼前一亮

### 输出
- 技术栈对比分析
- 优秀案例拆解
- 竞品分析矩阵
- 项目方向建议

## Stage 2 — 产品方案设计（Report Writing）
加载技能：`report-writing`

基于Stage 1的研究成果，撰写完整报告，包括：
1. 执行摘要
2. 技术栈深度分析（MCP、LangChain/LangGraph、记忆管理等）
3. 竞品与案例分析
4. 产品方案设计（核心功能、架构设计、技术选型）
5. 落地路线图（MVP → 完整产品）
6. 秋招展示策略

### 输出
- 完整的Markdown报告

## Stage 3 — 文档格式化（Artifact Production）
加载技能：`docx`

将Markdown报告转换为专业的Word文档，便于打印和展示。

### 输出
- .docx 格式最终报告
